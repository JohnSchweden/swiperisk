1. In pressureAudio.ts: Add `playUnlockPulse(ctx: AudioContext): void` — creates oscillator, connect, start(0), stop(0.05) — all synchronous. No await, no Promise. Must run in same call stack as touch.

2. In usePressureAudio.ts: Change resumeOnFirstGesture so that when user touches:
   - If ctx.state === "suspended": ctx.resume() then immediately playUnlockPulse(ctx) — both in same synchronous handler
   - The resume() returns a Promise; we cannot await. Instead: call playUnlockPulse(ctx) synchronously in the handler. Chrome may allow the first start() to "unlock" if we're in the gesture. If resume is required first, try: resume().then(() => playUnlockPulse(ctx)) — the .then runs as microtask, which might still be within the "sticky" gesture window on some browsers.

3. Fallback if .then is too late: Create the AudioContext ONLY on first touch, not on mount. Store a "ready" ref. On first touchend/click: create context, call resume(), playUnlockPulse. Then set ready. When hasHighPressure and ready, start heartbeat.

4. Prefer simplest fix first: resume().then(() => playUnlockPulse(ctx)) in the gesture handler. Test on Android.
