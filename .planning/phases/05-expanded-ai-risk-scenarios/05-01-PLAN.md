<task type="auto">
  <name>Task 3: Create no-win penalty validation test (with balance ratio)</name>
  <files>tests/data/card-penalties.test.ts</files>
  <action>
Create test file that validates no-win structure AND penalty balance:

1. **Test: "Both outcomes have non-zero penalties"** — For each card:
   - onLeft penalties: |hype| > 0 OR heat > 0 OR fine > 0
   - onRight penalties: |hype| > 0 OR heat > 0 OR fine > 0
   At least one of {hype, heat, fine} must be non-zero on EACH outcome. Fail if either is 0/0/0.

2. **Test: "Penalty balance ratio within 0.3-3.0x" (Issue #9)** — For each card:
   - Calculate totalLeft = abs(hype) + heat + (fine / 1_000_000)
   - Calculate totalRight = abs(hype) + heat + (fine / 1_000_000)
   - Ratio = totalLeft / totalRight (or inverse, whichever > 1)
   - FAIL if ratio > 3.0 (one outcome is 3x+ worse — not a real dilemma)
   - WARN if ratio > 2.5 (borderline — review manually)
   This catches lopsided "fake dilemmas" where one choice is obviously better.

3. **Test: "Fine values in range"** — Fine should be in range $0–$50M (Phase 03 baseline). Outcomes with fine=$0 must have heat > 20 or |hype| > 15 to ensure some cost.

4. **Test: "Violation text is meaningful"** — Non-empty, non-generic. Warn if single word or "Violation"/"Fine"/"Bad".

5. **Test: "Lesson explains tradeoff, not prescribes"** — Soft check: scan for "always," "never," "should" (prescriptive). Warn, don't fail.

Penalty ranges and checklist: See 05-CONTEXT.md "Card Design Checklist" and "Penalty Ranges".
