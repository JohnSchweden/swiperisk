<objective>
Fix the outcome image architecture to use label-based keys instead of direction-based keys.

The user identified a critical UX issue: swipe directions are arbitrary UI conventions, but the actual decisions are represented by the specific labels shown on cards (e.g., "Name the engineer" vs "Take the blame"). Currently, outcomes are keyed as `${incidentSlug}-left` and `${incidentSlug}-right`, which ties them to swipe direction. This is wrong because:

1. The same label might appear on different swipe directions in different contexts
2. Users make decisions based on labels, not directions
3. Outcome feedback (images, audio, text) should be tied to the specific decision, not the swipe

Purpose: Align outcome images with decision semantics (labels) rather than UI mechanics (swipe direction)
Output: Updated imageMap.ts, tests, and generation script using label-based outcome keys
</objective>

<gap_analysis>
**Critical Gap Identified:**

Current architecture ties outcomes to swipe direction:
- OUTCOME_IMAGES keys: `${incidentSlug}-left`, `${incidentSlug}-right`
- getOutcomeImagePath(incidentSlug, "LEFT" | "RIGHT")

Desired architecture ties outcomes to decision labels:
- OUTCOME_IMAGES keys: `${incidentSlug}-${labelSlug}`
- getOutcomeImagePath(incidentSlug, labelSlug)

**Example:**

Card: "GitHub Copilot RCE CVE-2025-53773"
- onLeft: { label: "Take the blame", lesson: "Taking accountability..." }
- onRight: { label: "Name the engineer", lesson: "Throwing team members..." }

Current (WRONG):
- Outcomes: `github-copilot-rce-cve-2025-53773-left`, `github-copilot-rce-cve-2025-53773-right`
- Tied to swipe direction

Desired (CORRECT):
- Outcomes: `github-copilot-rce-cve-2025-53773-take-the-blame`, `github-copilot-rce-cve-2025-53773-name-the-engineer`
- Tied to specific decision labels

**Files to Modify:**

1. data/imageMap.ts — Complete rewrite of OUTCOME_IMAGES and getOutcomeImagePath
2. data/index.ts — Export new slugifyLabel helper
3. tests/data/image-map.test.ts — Update tests for label-based outcomes
4. scripts/generate-images.ts — Update outcome extraction and prompt generation

**Impact on Phase 14:**

This change affects how Phase 14 UI looks up outcome images:
- OLD: `getOutcomeImagePath(incidentSlug, swipeDirection)`
- NEW: `getOutcomeImagePath(incidentSlug, labelSlug)`

The UI will need the card's label to look up the correct outcome image, which is correct because the UI displays the label to the user.
