    Create `data/imageMap.ts` following the project's established `Record<EnumType, T>` pattern (see archetypes.ts, deathEndings.ts for reference).

    The file must:

    1. **Export a `slugifyIncident` function** that converts incident names to URL-safe slugs:
       ```typescript
       export function slugifyIncident(incident: string): string {
         return incident
           .toLowerCase()
           .replace(/[^a-z0-9]+/g, "-")
           .replace(/^-|-$/g, "");
       }
       ```

    2. **Export type aliases** (in imageMap.ts, not types.ts):
       ```typescript
       export type IncidentSlug = string;
       /** Keys like `samsung-chatgpt-code-leak-left` — see 13-CONTRACT.md */
       export type OutcomeImageKey = string;
       ```

    3. **Build INCIDENT_IMAGES dynamically** for the **pilot scope only**: iterate **`ROLE_CARDS[RoleType.HEAD_OF_SOMETHING]`** (or the HOS deck export), not every role. Extract unique `realWorldReference.incident` values from those cards only.
       ```typescript
       import { ROLE_CARDS } from "./cards";
       import { RoleType } from "../types";

       const hosCards = ROLE_CARDS[RoleType.HEAD_OF_SOMETHING];
       const uniqueIncidents = new Set<string>();
       for (const card of hosCards) {
         if (card.realWorldReference?.incident) {
           uniqueIncidents.add(card.realWorldReference.incident);
         }
       }

       export const INCIDENT_IMAGES: Record<string, string> = Object.fromEntries(
         [...uniqueIncidents].map((incident) => {
           const slug = slugifyIncident(incident);
           return [slug, `/images/incidents/${slug}.webp`];
         })
       );
       ```
       **Expansion:** switch the source set to all `ROLE_CARDS` values (or additional roles) when maps grow — document the toggle in code comment. Pilot ~18 slugs; full deck ~118. Cards sharing the same incident string share one path.

       **Slug collision:** add a unit test or assert in build: if two distinct incident strings slugify to the same slug, throw or fail CI (silent overwrite risk).
