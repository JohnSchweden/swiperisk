**Verification Checklist (for each card sampled):**

**Card Authenticity:**
- [ ] Card narrative references 2024-2025 incidents (feels grounded in reality, not fiction)
  - Example: "GitHub Copilot CVE-2025-53773" or "EU AI Act compliance deadline" or "70+ copyright lawsuits by 2025"
  - Bad: "Imagine a super weird AI incident happened..."
- [ ] Role-specific context mentioned (not generic to all roles)
  - Example: "CFO would worry about budget overrun" OR "CTO would worry about timeline slip"
  - Bad: "Your company has AI. What do you do?"
- [ ] Incident framing is unique per role (same incident, different organizational lens)
  - Example: Same prompt injection incident should feel different for Finance (liability) vs. Dev (security patch)

**No-Win Structure:**
- [ ] Both outcomes carry visible costs (hype, heat, or fine non-zero)
  - Good: LEFT outcome hype +40 / heat 0 / fine 0, RIGHT outcome hype -30 / heat +60 / fine $2M
  - Bad: LEFT outcome hype 0 / heat 0 / fine 0, RIGHT outcome hype +50 / heat +50 / fine $5M (LEFT is obviously better)
- [ ] Lesson text explains the tradeoff, not prescribes the "right" choice
  - Good: "Timeline slip is the cost of security. You chose speed."
  - Bad: "Always choose security first. You failed."
- [ ] Choice feels morally ambiguous (reasonable people could choose either way)
  - Example: "Retrain the model (expensive, slow) vs. accept accuracy gap (risky, fast)" — both feel like legitimate business decisions

**Personality Feedback:**
- [ ] All 3 personalities present and distinct
  - ROASTER: Cynical, sarcastic, critical. "Hope that was worth it." / "We're so doomed."
  - ZEN_MASTER: Wisdom, acceptance, metaphorical. "The path you chose has a cost." / "Patience yields strength."
  - LOVEBOMBER: Enthusiastic, supportive, casual. "Besties!" / "You're doing great!" (even on bad outcomes)
- [ ] Feedback is outcome-specific (ROASTER says different things on LEFT vs. RIGHT)
  - Example: ROASTER on "Patch quickly" might be "Known gap, future lawsuit." ROASTER on "Rewrite properly" might be "Missed launch, but auditors sleep well."
  - Bad: Same ROASTER feedback on both outcomes
- [ ] Feedback is contextual (references the specific choice, not generic)
  - Good: "Copilot integration now crippled, but at least you're secure."
  - Bad: "You made a choice."

**Game Flow:**
- [ ] No crashes, errors, or UI breaks
- [ ] Cards appear in random order (shuffle working)
- [ ] Feedback overlay displays correctly (text readable, penalties visible)
- [ ] Card text is clear and readable (not truncated, formatting OK)
- [ ] Transitions between cards smooth (no lag, timing feels right)

**Overall Engagement:**
- [ ] Cards feel immersive (you can imagine being in that situation)
- [ ] Cards make you think (not trivial choices)
- [ ] Personality feedback makes you smile or laugh (not cringey)
- [ ] You want to play more cards (not bored after 3 cards)

**If you find issues, note them below:**
- [ ] Card ID and issue (e.g., "prompt_injection_dev_1: feedback on LEFT is generic, should mention security timeline")
- [ ] Category (authenticity, no-win structure, personality, flow, engagement)
- [ ] Severity (blocker: unplayable / major: awkward / minor: typo)

**Summary after playing:**
- How many cards felt authentic? (out of 10-15 sampled)
- Did any card feel like a logic puzzle (one obvious choice) vs. a genuine dilemma?
- Did personality feedback make you laugh/engage?
- Any crashes or UI issues?
