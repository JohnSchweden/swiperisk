## **How Audio Files Will Be Generated**

### **Reusing Existing Pipeline**

**YES - This plan reuses the existing audio generation infrastructure from v1.1:**

1. **scripts/generate-voice.ts** - Main TTS generation script using Gemini API
   - Uses Google Gemini's text-to-speech API
   - Voice assignments: "Kore" (Roaster), "Puck" (ZenMaster), "Enceladus" (Lovebomber)
   - Generates 24kHz mono WAV files (PCM 16-bit)

2. **scripts/generate-all.ts** - Batch generation orchestrator
   - Iterates through all scripts and generates files
   - Applies consistent naming conventions
   - Validates output files

3. **services/radioEffect.ts** - Post-processing pipeline
   - Applies "radio" distortion effects (bandpass filter, noise, crackle)
   - Adds Quindar intro tones (NASA-style beeps)
   - Normalizes volume to 0.4 (40%)

### **Generation Process**

```
Text Script → Gemini TTS API → Raw WAV → Radio Effects → Final WAV
```

**For each audio file:**
1. Load text script from plan documentation
2. Call Gemini TTS with personality-specific voice
3. Save raw WAV to temp directory
4. Apply radio effects via AudioWorklet/AudioContext
5. Export final WAV to public/audio/voices/{personality}/
6. Validate file size and format

### **File Naming Convention**

```
public/audio/voices/{personality}/{trigger}.wav

Examples:
- public/audio/voices/roaster/death_bankrupt.wav
- public/audio/voices/zenmaster/death_kirk.wav
- public/audio/voices/lovebomber/death_congress.wav
```

### **KIRK Special Handling**

KIRK death audio uses a hybrid approach:
1. **Base layer:** Existing synthesized glitch from services/kirkAudio.ts
   - playKirkGlitchTone() - subtle corruption tone
   - playKirkCrashSound() - intense crash sound
2. **Voice layer:** TTS with heavy distortion applied
   - Same generate-voice.ts pipeline
   - Extra radio effects (more distortion, shorter delays)
3. **Combined:** Both play simultaneously in DebriefPage1Collapse

### **What You'll Need to Run**

```bash
# Set Gemini API key
export GEMINI_API_KEY=your_key_here

# Run batch generation
bun run scripts/generate-all.ts

# Or generate specific files
bun run scripts/generate-voice.ts --voice Kore --text "Liquidated..." --output death_bankrupt.wav
```

### **Validation Steps**

After generation:
1. File format check: `file public/audio/voices/*/*.wav` (should show RIFF WAVE)
2. File size check: All files > 50KB (valid audio, not empty)
3. Audio playback test: Listen to each file for clarity and radio effects
4. Integration test: Verify triggers work in game
