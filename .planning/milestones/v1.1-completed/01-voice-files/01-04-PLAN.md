<feature>
  <name>Voice Playback System</name>
  <files>services/voicePlayback.ts, tests/voicePlayback.spec.ts</files>
  <behavior>
Voice playback system must:

1. **Load voice file**: Given a personality and trigger, load the correct WAV file
   - Input: {personality: 'roaster' | 'zenmaster' | 'lovebomber', trigger: 'onboarding' | 'victory' | 'failure' | 'feedback_paste' | 'feedback_debug' | 'feedback_install' | 'feedback_ignore'}
   - Output: Promise that resolves when file is loaded, rejects with narrative error if file missing

2. **Play voice**: Play the loaded audio
   - Input: None (uses previously loaded audio)
   - Output: Promise that resolves when playback starts

3. **Stop voice**: Stop any currently playing audio
   - Input: None
   - Output: void

4. **Error handling**: If audio file fails to load/play, return error message framed as in-game narrative
   - Roaster error: "V.E.R.A. voice module malfunctioned"
   - Zen Master error: "The silence of the spreadsheets is deafening"
   - Lovebomber error: "OMG the audio broke!! But we still love you!!"

Voice trigger types:
- 'onboarding' - played when user selects personality
- 'victory' - played when user wins a stage
- 'failure' - played when user loses a stage
- 'feedback_paste' - played after pasting (Roaster only)
- 'feedback_debug' - played after debugging (Roaster only)
- 'feedback_install' - played after installing (Roaster only)
- 'feedback_ignore' - played after ignoring warning (Roaster only)
  </behavior>
  <implementation>
Implementation approach:

1. Create VoiceTrigger type in types.ts
2. Create services/voicePlayback.ts with:
   - loadVoice(personality, trigger): Promise<void>
   - playVoice(): Promise<void>
   - stopVoice(): void
   - Preload AudioContext from existing geminiService.ts pattern
   - Use fetch to load WAV files from public/audio/voices/
   - Decode using AudioContext.decodeAudioData()
   - Store loaded buffer for playback

3. Create tests/voicePlayback.spec.ts with Playwright tests
   </implementation>
</feature>
