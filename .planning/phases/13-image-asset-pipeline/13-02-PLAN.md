<task type="auto">
  <name>Task 1: Create image generation pipeline script with auto-prompt generation</name>
  <files>scripts/generate-images.ts, package.json</files>
  <action>
    Install sharp as a dev dependency:
    ```bash
    bun add -D sharp @types/sharp
    ```

    Create `scripts/generate-images.ts` following the existing `scripts/generate-voice.ts` pattern.

    The script should:

    1. **Read GEMINI_API_KEY from env** — exit with clear error if missing (unless --dry-run)

    2. **Import card decks and imageMap utilities**:
       ```typescript
       import { slugifyIncident } from "../data/imageMap";
       // Import all card arrays from data/cards/
       ```

    3. **Extract incidents from card data**:
       - **Generation queue:** When `--scope hos`, only **Head of Something** cards feed incident/outcome tasks (matches **13-CONTRACT** pilot). Other scopes use the appropriate deck filter or full map.
       - **Optional:** Scan all `ROLE_CARDS` only for **logging** (shared-incident report across roles), not for pilot task enumeration.
       - For each selected card with `realWorldReference`, extract: `incident`, `date`, `outcome`; dedupe by slug; track roles if reporting.
       - Build a `Map<string, IncidentEntry>` keyed by slug for the active scope.

    4. **Auto-generate prompts from card data** (no hardcoded prompt library):
       - For each unique incident, generate a prompt using an art direction template:
         ```typescript
         function generateIncidentPrompt(entry: IncidentEntry): string {
           return `Satirical editorial photograph for a corporate tech magazine. ` +
             `Scene depicts the real-world incident: "${entry.incident}" (${entry.date}). ` +
             `Context: ${entry.outcome}. ` +
             `Style: Hyper-realistic corporate photography with subtle AI artifacts — ` +
             `extra fingers, warped text on screens, impossible reflections. ` +
             `The image should feel like an AI-generated stock photo that is almost right. ` +
             `Fictional, stylized, editorial illustration.`;
         }
         ```
       - Similar template functions for outcome, archetype, and death prompts
       - **Outcome prompts:** per **incident slug + direction**, using canonical `onLeft.lesson` / `onRight.lesson` + `realWorldReference` context (not eight consequence-type labels)
       - Archetype prompts use archetype name + traits from data/archetypes.ts
       - Death prompts use death type name + description from data/deathEndings.ts

    5. **Accept CLI args**:
       - `bun scripts/generate-images.ts` — generate all categories (respects pilot vs full per map)
       - `bun scripts/generate-images.ts --scope hos` — **pilot:** HOS incidents + HOS outcome pairs only
       - `bun scripts/generate-images.ts --scope incidents` — incident images only
       - `bun scripts/generate-images.ts --scope outcomes` — outcome pairs only (`-left`/`-right` filenames)
       - `bun scripts/generate-images.ts --scope archetypes` — archetype images only
       - `bun scripts/generate-images.ts --scope deaths` — death images only
       - `bun scripts/generate-images.ts --dry-run` — print tasks/prompts without API calls
       - `bun scripts/generate-images.ts --export-prompts` — write markdown under `scripts/prompts/` (no API)
       - `bun scripts/generate-images.ts --force` — regenerate even if file exists
       - `bun scripts/generate-images.ts --model …` — override model
       - `bun scripts/generate-images.ts --slug <incident-slug>` — single reference case (incident + both outcomes as needed)

    6. **Category-to-directory mapping:**
       ```typescript
       const CATEGORY_DIR: Record<string, string> = {
         incident: "public/images/incidents",
         outcome: "public/images/outcomes",
         archetype: "public/images/archetypes",
         death: "public/images/deaths",
       };
       ```

    7. **Generate each image:**
       - Use `ai.models.generateContent` with `responseModalities: ["IMAGE"]`
       - Model default: `gemini-2.5-flash-image` (overridable via --model flag)
       - Set safety settings to `BLOCK_ONLY_HIGH` for all 4 categories
       - Extract base64 image data from response

    8. **Convert PNG to WebP via sharp:**
       - Resize: max **800px** width (withoutEnlargement: true)
       - WebP quality: **75**
       - Log output file size

    9. **Save to correct directory:** `{CATEGORY_DIR[category]}/{slug}.webp`

    10. **Rate limiting:** 7-second delay between requests (stays under 10 IPM Tier 1 limit)

    11. **Error handling:**
        - On safety filter block (finishReason: "SAFETY"): log warning, continue to next prompt
        - On 429 rate limit: wait 60 seconds, retry once
        - On other error: log, continue to next prompt
        - At end: print summary (generated, skipped, failed, shared-across-roles)

    12. **Skip existing:** If output file already exists and --force not passed, skip it (incremental regeneration)

    13. **Report cross-role sharing:** After generation, log which incidents are shared across multiple roles
        ```
        Shared incidents:
          samsung-chatgpt-code-leak: HEAD_OF_SOMETHING, SOFTWARE_ENGINEER, …
          ...
        ```

    14. **Human checkpoint (13-CONTEXT):** After the **first** successful image is written in a non-dry-run batch, **pause** with a readline prompt (print path); user confirms before continuing — saves quota if art direction is wrong.

    Add scripts to package.json (example):
    ```json
    "generate:images": "bun scripts/generate-images.ts",
    "generate:images:pilot": "bun scripts/generate-images.ts --scope hos --dry-run"
    ```
