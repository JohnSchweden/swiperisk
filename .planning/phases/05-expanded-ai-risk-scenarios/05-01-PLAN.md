  <name>Task 3: Create no-win penalty validation test</name>
  <files>tests/data/card-penalties.test.ts</files>
  <action>
Create test file that validates no-win structure (both outcomes carry penalties):

1. **Test: "Both outcomes have non-zero penalties"** — For each card, check:
   - onLeft penalties: |hype| > 0 OR heat > 0 OR fine > 0
   - onRight penalties: |hype| > 0 OR heat > 0 OR fine > 0
   At least one of {hype, heat, fine} must be non-zero on EACH outcome. Fail if either outcome has 0/0/0 (that's a puzzle, not a dilemma).

2. **Test: "Penalty magnitudes are comparable"** — For each card, calculate total cost: abs(hype) + heat + (fine / 1_000_000). Compare left vs right. If one path is 10x cheaper than the other, warn (soft check). Example: LEFT is 5/10/0 (total 15), RIGHT is 40/40/5000000 (total 5040000). That's wildly imbalanced.

3. **Test: "Fine values in historical range"** — Check fine values against Phase 03 baseline. Fine should be in range $0–$5M (from research docs). If fine is $0 (no legal penalty) on an outcome, that outcome must have heat > 50 or |hype| > 30 to ensure some cost.

4. **Test: "Violation text is meaningful"** — For each outcome, violation field must be non-empty and non-generic. Examples: "Trade Secret Misappropriation (DTSA)" (good), "None - Secure coding practices" (good), "Bad stuff" (bad). Soft check: if violation is single word or generic like "Violation" or "Fine," warn for review.

5. **Test: "Lesson explains tradeoff, not prescribes"** — Soft check: scan lesson text for words like "always," "never," "should" (prescriptive). If found, warn that lesson might be teaching instead of reflecting. This is a style check, not a fail condition.

Vitest imports: `import { describe, it, expect } from 'vitest'`
Test data: Load ROLE_CARDS

Ranges: Based on Phase 03 observed (research docs, line 596): "heat 0-100, fine $0-50M"
