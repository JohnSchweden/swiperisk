Card interface (from types.ts):
```typescript
export interface Card {
  id: string;
  source: AppSource;
  sender: string;
  context: string;
  storyContext?: string;
  text: string;
  onRight: { label, hype, heat, fine, violation, feedback, lesson };
  onLeft: { label, hype, heat, fine, violation, feedback, lesson };
}

// Personality voices
export enum PersonalityType {
  ROASTER = "ROASTER",
  ZEN_MASTER = "ZEN_MASTER",
  LOVEBOMBER = "LOVEBOMBER",
}
```

Example card structure (from data/cards/chief-something-officer.ts and vibe-coder.ts):
- Both outcomes (onRight, onLeft) always present
- Both carry hype, heat, fine (can be positive or negative)
- Both have lesson field (explain tradeoff, not winner)
- Feedback object requires all 3 personality keys
- Real 2024-2025 incidents sourced

  <files>tests/data/card-penalties.test.ts</files>
  <behavior>

    - Test: For each card, both onRight and onLeft have non-zero heat (absolute value >= some threshold)
    - Test: Card is not free (one path has zero heat + zero hype + zero fine = no-win violated)
    - Test: Penalty magnitude comparable between left/right (e.g., no path is obviously dominant)
    - Test: fine >= 0 (never negative; can be 0 for compliant path)
    - Edge case: Detects "dominant strategy" pattern (one outcome hype=0, heat=0, fine=0 while other suffers; invalid)
  </behavior>
  <action>
Create test file `tests/data/card-penalties.test.ts` using Vitest. Validates no-win constraint: both outcomes must carry penalties.

Import card arrays (same as task 1). For each card:
1. Calculate penalty magnitude as: `Math.abs(hype) + Math.abs(heat) + (fine / 1000000)` (normalize fine to millions for comparison)
2. Validate both onRight and onLeft have penalty magnitude > 0 (at minimum, some cost to both paths)
3. Validate neither path is "free": if hype=0 AND heat=0 AND fine=0, fail with message "Card {id} has free path, violates no-win constraint"
4. Optional: warn if one path has 5x+ penalty of other (suggests dominant strategy)

Test suite name: `describe('Card Penalties — No-Win Balance')`
Individual tests: `it('validates {role} penalties are comparable')`

**Why:** Pitfall 1 from RESEARCH.md — cards with dominant strategies become logic puzzles instead of character tests.
