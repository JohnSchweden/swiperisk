**Research Pattern Applied:** Clipboard API with execCommand Fallback (RESEARCH.md Pattern 4)

**Research Findings:**
- Clipboard API requires secure context (HTTPS) and transient user activation
- iOS Safari can fail with `navigator.clipboard.writeText()` outside trusted gesture chain
- execCommand fallback works synchronously within user gesture on iOS
- Touch targets need min 44px per iOS HIG/WCAG 2.5.5

Replace the inline `onClick` on the "Copy game link" button with a research-validated iOS-aware handler.
**Note:** Android already works with `navigator.clipboard`. This fix specifically addresses iOS Safari per RESEARCH.md.

1. Define a `handleCopyLink` function (above the return statement, inside the component):
```typescript
const handleCopyLink = async () => {
  const url = window.location.href;
  // Modern Clipboard API — works on desktop and Android (RESEARCH.md primary)
  if (navigator.clipboard && typeof navigator.clipboard.writeText === "function") {
    try {
      await navigator.clipboard.writeText(url);
      return;
    } catch {
      // iOS Safari may reject — fall through to execCommand fallback (RESEARCH.md)
    }
  }
  // execCommand fallback for iOS Safari (RESEARCH.md Pattern 4)
  // Works synchronously within user gesture, satisfying Safari's requirements
  const el = document.createElement("textarea");
  el.value = url;
  el.style.position = "fixed";
  el.style.left = "-9999px";
  el.style.top = "-9999px";
  document.body.appendChild(el);
  el.focus();
  el.select();
  try {
    document.execCommand("copy");
  } finally {
    document.body.removeChild(el);
  }
};
```

2. Update the button to use the handler and add `min-h-[44px]` per RESEARCH.md touch target standards:
```tsx
<button
  type="button"
  data-testid="copy-game-link-button"
  onClick={() => void handleCopyLink()}
  className="px-4 py-2 text-xs font-bold tracking-wide border border-slate-600 text-slate-400 hover:border-cyan-500/40 hover:text-cyan-300 transition-colors duration-200 min-h-[44px]"
>
  Copy game link
</button>
```

**Research Insight:** The `textarea` fallback works synchronously within the user gesture on iOS, satisfying Safari's clipboard permission requirements. Android continues to use the modern Clipboard API successfully. This follows the RESEARCH.md "Don't Hand-Roll" guidance: use Clipboard API + fallback, not execCommand only.
