<objective>
Add automatic haptic triggers for:
1. When heat crosses into critical (heat >= 70)
2. When countdown timer expires (auto-resolve)

Currently haptics only fire on swipe (onBeforeSwipe). User expects automatic vibration when critical state triggers or timer expires.
</objective>

<context>

The UAT test expects haptics in these scenarios:
- "when entering critical state" → heat >= 70 triggers isCritical
- "when the timer expires" → countdown reaches 0

Current code only vibrates on swipe. Need to add haptics for automatic state changes.

</context>

<tasks>

## Task 1: Add onCriticalChange callback to useIncidentPressure

In `hooks/useIncidentPressure.ts`:

1. Add `onCriticalChange?: (isCritical: boolean) => void` parameter to the hook
2. Use a useEffect to detect when `isCritical` transitions from false → true
3. When transition to true occurs, call `onCriticalChange(true)` if provided

This detects when heat crosses the critical threshold.

## Task 2: Add onExpire callback to useCountdown

In `hooks/useCountdown.ts`:

1. Add `onExpire?: () => void` parameter to the hook
2. In the effect, when natural expiry is detected (count 1→0), call `onExpire()` before calling `onComplete()`

This triggers when countdown naturally reaches zero.

## Task 3: Wire up automatic haptics in App.tsx

In `App.tsx`, when using the hooks:

1. For useIncidentPressure:
   - Pass `onCriticalChange={(isCritical) => { if (isCritical) triggerHaptic(); }}`
   - Define `triggerHaptic` helper that checks navigator.vibrate availability

2. For useCountdown:
   - Pass `onExpire={() => { triggerHaptic(); }}`
   - Use same `triggerHaptic` helper

3. Create shared `triggerHaptic` function:
```typescript
const triggerHaptic = useCallback(() => {
  if (
    typeof navigator !== "undefined" &&
    "vibrate" in navigator &&
    typeof navigator.vibrate === "function"
  ) {
    navigator.vibrate([50, 30, 50]);
  }
}, []);
```

Note: Both callbacks may fire in same incident (critical + timer expiry). That's OK — vibrate calls queue or replace.

</tasks>
